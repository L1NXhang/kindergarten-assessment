# 幼儿园幼儿发展评估网页系统 — 实现计划

## 摘要

开发一个基于轶事记录法的3-6岁幼儿发展评估网页系统。Python Flask + Vue 3 SPA + SQLite，本地部署局域网访问。核心功能：轶事记录（自动识别幼儿姓名）→ DeepSeek AI 生成阶段性评估报告 → 历史评估对比。无需登录界面，教师信息从记录中自动提取。

## 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 后端 | Python 3 + Flask 3.1 | REST API |
| ORM | Flask-SQLAlchemy | 参数化查询防SQL注入 |
| 数据库 | SQLite（单文件） | `instance/assessment.db` |
| 前端 | Vue 3 + Vite 6 + Tailwind CSS 4 | SPA |
| 状态管理 | Pinia | |
| 图表 | Chart.js + vue-chartjs | 雷达图/趋势图 |
| AI | DeepSeek API（openai SDK兼容） | 报告生成 |
| 安全 | Flask-WTF CSRF + Flask-Talisman | 安全头/CSP |
| 部署 | Flask托管前端静态文件 | `0.0.0.0:5000` |

### 核心页面：AnecdoteFormView（轶事记录填写）

这是系统最核心的页面，设计原则是**老师能最快速度完成录入**：

- **表单区**（左侧/上方）：
  - 班级选择（下拉，记住上次选择）
  - 观察地点（预设标签按钮组：户外操场/建构区/阅读区/美工区/角色区/科学区/生活活动/集体教学）

  - 活动类型（自由游戏/集体教学/区域活动/生活活动/户外活动）
  - 观察内容（大文本框，最少20字）
- **实时识别区**（文本框下方）：
  - 输入时防抖300ms调用姓名识别API
  - 识别到的幼儿姓名以彩色标签展示
  - 支持确认/移除/手动添加
- **教师信息**：自动从"XX老师"模式提取，也可手动输入

## 五、核心算法

### 5.1 姓名识别算法

采用**班级内精确匹配**策略（不使用NLP NER，因为幼儿名单已知且有限）：

```
输入：content（观察文本）, class_id（班级ID）
输出：{children: [{id, name, confidence, positions}], teacher: {id, name}}

1. 从children表获取该班级所有幼儿姓名 → name_set
2. 精确匹配：遍历name_set，re.finditer每个姓名在content中的位置
   - 匹配成功 → confidence = 1.0
3. 模糊容错（可选增强）：
   - 去掉"小"前缀、处理空格分隔
   - 模糊匹配 → confidence = 0.7
4. 教师提取：正则匹配"XX老师"模式
5. 按首次出现位置排序返回
```

### 5.2 AI报告生成（DeepSeek API）

- 模型：`deepseek-chat`（DeepSeek-V3）
- SDK：`openai` Python包（兼容接口，base_url=`https://api.deepseek.com`）
- temperature=0.3（保证一致性）
- 异步生成：后端threading.Thread，前端轮询进度

**三种报告Prompt**（详见 `prompts/` 目录）：
1. **个人发展报告**：按《指南》五大领域分析，输出发展水平(A/B/C/D) + JSON评分 + 优势/建议
2. **班级汇总报告**：班级整体发展状况 + 共性问题 + 关注幼儿清单
3. **教育反思与支持策略**：实践反思 + 个别化策略 + 环境调整 + 家园共育

### 5.3 Excel导入

- 使用 `openpyxl` 读取
- 自动识别表头映射（支持"姓名"/"学生姓名"/"幼儿姓名"等变体）
- 必填列：姓名；可选列：性别、出生日期、备注
- 导入预览 + 去重检查 + 错误行提示

## 六、安全设计

### Flask后端
- **SQL注入**：全部使用SQLAlchemy ORM参数化查询
- **CSRF**：Flask-WTF CSRFProtect（所有POST/PUT/DELETE）
- **安全头**：Flask-Talisman（CSP、X-Content-Type-Options、X-Frame-Options）
- **文件上传**：扩展名白名单(.xlsx/.xls) + Magic Number验证 + 16MB大小限制
- **SECRET_KEY**：从环境变量读取，不硬编码
- **Session**：HTTPOnly=True, SameSite=Lax（局域网HTTP，Secure=False）
- **调试模式**：生产环境强制关闭

### Vue前端
- **XSS防护**：不使用v-html，AI报告用marked+DOMPurify安全渲染
- **Secrets**：DeepSeek API Key仅存后端.env，前端不暴露
- **构建**：vite build生产构建，不暴露源码
- **输入验证**：前端基本验证 + 后端完整验证

## 十、假设与决策

1. **无登录系统**：局域网内部使用，信任环境，不设认证
2. **教师自动提取**：从轶事记录文本中识别"XX老师"模式，无需预先导入教师名单
3. **姓名识别用精确匹配**：班级内名单已知且有限，无需NLP NER
4. **AI报告异步生成**：使用threading而非Celery，适合本地单机部署
5. **前端SPA**：用户选择Vue/React SPA，选择Vue 3
6. **DeepSeek API**：国产模型，中文能力强，价格低
7. **SQLite WAL模式**：启用WAL以支持并发读（局域网多设备同时访问）

## 十一、验证步骤

1. 启动后端，验证所有API端点返回正确响应
2. 创建班级 → Excel导入幼儿 → 验证数据库记录
3. 填写轶事记录 → 验证姓名自动识别准确率
4. 创建评估任务 → 验证AI报告生成（需配置DeepSeek API Key）
5. 生成两次评估 → 验证历史对比功能
6. 备份/恢复数据库 → 验证数据完整性
7. 从另一台设备访问 → 验证局域网访问
8. 安全审计：检查SQL注入、XSS、CSRF、文件上传安全
